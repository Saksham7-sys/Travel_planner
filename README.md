## 📌 What is TripTrekker?

TripTrekker is a full-stack travel route optimization app. You pick up to **12 tourist spots** in Meghalaya — waterfalls, caves, living root bridges, national parks — and the app computes the **optimal visiting order** that minimizes total road distance, then draws the route on an interactive map.

It solves the **Travelling Salesman Problem (TSP)** — an NP-hard combinatorial optimization problem — using a C++ Branch-and-Bound algorithm for exact solutions.

---

## ✨ Features

- 🧠 **Exact TSP solver** — Branch-and-Bound with lower-bound pruning (no approximation)
- 📍 **29 curated Meghalaya locations** — waterfalls, lakes, caves, bridges, cities, parks
- 🗺️ **Interactive Leaflet.js map** — click pins or cards to select spots
- 🎨 **Category filtering** — filter by Falls, Lakes, Caves, Bridges, Parks, etc.
- 📏 **Haversine distance matrix** — straight-line distances × 1.5 terrain factor for hilly roads
- ⚡ **Animated route polyline** — see the optimized path drawn on the dark map
- 🖥️ **No paid APIs** — runs fully offline once set up

## 🔬 How the Algorithm Works

### Travelling Salesman Problem (TSP)
Given N locations, find the shortest route that visits every location exactly once and returns to the start. This is **NP-hard** — brute force checks N! routes (12! = 479 million).

### Branch-and-Bound
Instead of checking every route, we:
1. Compute a **lower bound** for each partial path using the minimum outgoing edges (`firstMin` + `secondMin`) / 2
2. **Prune** any branch where `lower_bound + current_cost ≥ best_solution_so_far`
3. Recurse only on promising branches

This reduces the search space dramatically — 12 nodes solves in milliseconds.

### Distance Matrix
Distances are computed using the **Haversine formula** (great-circle distance between GPS coordinates), multiplied by **1.5** to account for Meghalaya's hilly terrain and winding roads.

```
d = 2R × arcsin(√(sin²(Δlat/2) + cos(lat1)·cos(lat2)·sin²(Δlng/2)))
road_distance ≈ d × 1.5
```
